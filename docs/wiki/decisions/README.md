# Decisions Index

Decisions are named `D-YYYYMMDD-<author>-<slug>.md`, for example `D-20260622-alex-money-as-integer-cents.md`. There is no sequential number and no "next id" to coordinate: a new decision uses today's date, the author handle, and a short slug. Two people on parallel branches never collide, and attribution is built into the name. The index table below is generated from the files by `scripts/decisions-index.sh`; do not hand-edit between the `<!-- index:* -->` markers.

Regenerate after adding or changing a decision (merge conflicts in the table resolve by re-running this):

```bash
bash scripts/decisions-index.sh         # rewrite the index table from the D-*.md files
bash scripts/decisions-index.sh --check # CI mode: exit non-zero if the index is stale
```

## Authoring criteria

- An open question is not a decision. Keep it in `.planning/threads/` first.
- Once the conclusion closes, copy `D-template.md` to `D-YYYYMMDD-<author>-<slug>.md` and run `scripts/decisions-index.sh` in the same change.
- `<author>` is the git author handle (lowercase). `<slug>` is a short English kebab-case phrase. Use today's date for `YYYYMMDD`.
- Fill in `status`, `date`, `decision_area`, `decision_subjects`, `decision_cluster`, `collapsed_to`, `supersedes`, `deciders`, and `context`.
- Record the options, rejected alternatives, impact, verification, and reversal conditions.
- After an important decision, run a Review Gate Quiz of up to three questions.

## Compaction criteria

- Do not create subfolders for classification. Classify with `decision_area`, `decision_subjects`, `decision_cluster`, and index views.
- Compact older decisions that group into one logical unit into a new consolidating decision.
- Do not delete or rename compacted source decisions. Hand the active SSOT over with `status: collapsed` and `collapsed_to: <id>` (the consolidating decision's id).
- The consolidating decision keeps the compacted sources in frontmatter, for example `supersedes: [D-20260101-alex-foo, D-20260115-sam-bar]`.
- A compaction is itself a new decision, so run `scripts/decisions-index.sh` afterward to regenerate the index.

## Status values

| Value | Meaning |
|---|---|
| `draft` | Draft. Conclusion not yet closed, or pending approval |
| `active` | Currently adopted decision |
| `pending` | Conclusion is close but on hold |
| `superseded` | Replaced by another decision |
| `collapsed` | Compacted into a larger consolidating decision. The active SSOT is the `collapsed_to` target |
| `deprecated` | No longer used but kept as a historical record |
